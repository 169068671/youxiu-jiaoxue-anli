代码造物数字工坊

📍 第 20 课：智造发布会（答辩、开源与反思）

答辩不是只展示成品，而是说明需求、原理、代码、迭代和证据。

本课要完成什么

  1. 用自己的话说清楚：答辩不是只展示成品，而是说明需求、原理、代码、迭代和证据。
  2. 看懂并计算：项目总评 = 创意20 + 技术25 + 打印20 + 成品15 + 文档10 + 答辩10
  3. 运行一段最小代码，完成：计算项目总分并检查是否超过100分

先把关系说成人话

数量关系： 项目总评 = 创意20 + 技术25 + 打印20 + 成品15 + 文档10 + 答辩10

怎么理解： 每一项都有证据：模型文件、代码、照片、记录和现场表达。

数字例子： 技术22、打印18、成品13、文档9、答辩9、创意18，总分89。

生活类比： 开源是把方法分享出去，让别人能理解、复现和改进。

已验证的课堂最小代码

下面的代码聚焦本课核心知识点，适用于带 Python 控制台的 FreeCAD 桌面版。

score = {"创意":18, "技术":22, "打印":18,
         "成品":13, "文档":9, "答辩":9}

total = sum(score.values())
print("项目总分：", total)
print("分数有效：", 0 <= total <= 100)

按这五步运行

  1. 打开FreeCAD，新建文档。
  2. 打开“视图 → 面板 → Python控制台”。
  3. 先原样运行代码，确认总分为89且显示“分数有效：True”。
  4. 修改一项得分，确认总分随之变化且不超过该项满分。
  5. 记录“改了什么、结果怎样、为什么会变”。
