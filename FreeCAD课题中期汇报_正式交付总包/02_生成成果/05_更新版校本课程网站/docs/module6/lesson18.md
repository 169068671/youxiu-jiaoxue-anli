# 📍 第 18 课：项目蓝图（需求分析与程序架构）

> 大型项目先拆需求、界面、数据和模型，再开始写代码。

## 本课要完成什么

1. 用自己的话说清楚：大型项目先拆需求、界面、数据和模型，再开始写代码。
2. 看懂并计算：项目 = 输入 + 处理规则 + 输出 + 测试
3. 运行一段最小代码，完成：写出项目需求清单

## 先把关系说成人话

**数量关系：** 项目 = 输入 + 处理规则 + 输出 + 测试

**怎么理解：** 先画结构图和伪代码，能减少边写边改造成的混乱。

**数字例子：** 参数化笔筒：输入尺寸→检查范围→生成模型→预览→导出STL。

**生活类比：** 盖房子先画蓝图，项目开发也先设计架构。

## 已验证的课堂最小代码

下面的代码聚焦本课核心知识点，适用于带 Python 控制台的 FreeCAD 桌面版。

```python
project = {
    "需求": "制作可打印的参数化笔筒",
    "输入": ["外半径", "壁厚", "高度"],
    "处理": ["检查参数", "生成外形", "挖空内腔"],
    "输出": ["FreeCAD模型", "STL文件"],
    "测试": ["尺寸正确", "壁厚大于1.2mm", "模型封闭"]
}

for stage, items in project.items():
    print(stage, "：", items)
```

## 按这五步运行

1. 打开FreeCAD，新建文档。
2. 打开“视图 → 面板 → Python控制台”。
3. 先原样运行代码，确认控制台输出需求、输入、处理、输出和测试清单。
4. 把项目需求改成自己的主题，并补充至少一项测试标准。
5. 记录“改了什么、结果怎样、为什么会变”。

## 常见问题

- 功能目标太多没有主线
- 界面与模型逻辑混在一起
- 没有测试异常输入

## 分层任务

### 基础任务

写出项目需求清单

### 进阶任务

画出输入—处理—输出流程

### 创意任务

确定小组分工和文件结构
